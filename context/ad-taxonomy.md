# 📌 Naming Taxonomy (Meta + All Platforms)

**Use this naming convention for every asset we create** (ads, organic posts, influencer/whitelisted content, boosted posts) across Meta and other platforms.

## Structure

```
[L1]_[L2]_[L3]_[L4]_[L5]_[L6]_[L7]_[L8]_[L9]_[L10]_[L11]_[L13]_[L14]_[L12]
```

✅ **Rule:** **L12 (Date & Version) must ALWAYS be the last item** in the string.

### Example

```
AD_FB-RL_FS_TOF_CONV_VDAY_VID-9x16_MA_1835_MNL_PIN_EN_LIFE_260210_001
```

## 🔹 Level 9: Age Range (numeric)

Use a **4-digit numeric range**: `AABB` = min age `AA` to max age `BB`.

Examples:
- `1835` = 18–35
- `2445` = 24–45
- `4565` = 45–65
- `1865` = 18–65+

**Rule:** always write as 4 digits (no dashes), e.g. `1835` not `18-35`.

## 🔹 Level 13: Language

Language code for the creative/copy.

| Code | Description |
| ---- | ----------- |
| EN | English (default for SG) |
| TL | Tagalog |
| TH | Thai |
| VI | Vietnamese |
| TL-EN | Tagalog + English mix (common for PH) |
| TH-EN | Thai + English mix (common for TH) |
| VI-EN | Vietnamese + English mix (common for VN) |

**Guideline by market:**
- **SG:** `EN`
- **PH:** `EN`, `TL`, or `TL-EN`
- **TH:** `TH` or `TH-EN`
- **VN:** `VI` or `VI-EN`

## 🔹 Level 12: Date & Version

| Component | Description |
| --------- | ---------------------------------- |
| YYMMDD | Date (e.g., 240520 = May 20, 2024) |
| ID | Version number (001, 002, etc.) |

**Format:** `YYMMDD_ID` (example: `260324_001`).

✅ **Rule:** this must ALWAYS be last.
