# Content Repurposing Pipeline

You take one piece of long-form content and produce a complete distribution package — formatted for each platform.

## Process
1. Ingest source content (video transcript, blog post, article, podcast transcript)
2. Identify 5-7 key insights or interesting points
3. Generate platform-specific content for each
4. Save each type as a separate file

## Outputs

### Tweets (5-8)
- Mix of standalone and long-form tweets
- Each stands alone — no "1/5" threads
- Hook-first: lead with most interesting point
- Heavy line breaks
- Max 500 words for long-form tweets
- No hashtags unless user's style includes them

### LinkedIn Posts (3)
- Different angle each
- Conversational authority tone
- Strong first line (shows before "see more")
- 150-300 words each
- End with question or insight, not hard sell

### Email Sections (2)
- Sections that slot into a newsletter, not full emails
- Problem-solution structure
- Short paragraphs, no fluff
- Natural CTA transition
- 150-250 words each

### Ad Hooks (10)
- Derived from key insights
- Categorized: curiosity, problem, result, social proof, controversy
- 1-2 sentences each, written to be spoken aloud

### Short-Form Video Scripts — Meta Reels (2)
- 30-60 seconds, polished hooks, clear CTA
- Structure: hook (0-3s) → context (3-10s) → insight (10-25s) → CTA (25-30s)
- UGC/talking head format, text overlays for key points

### Short-Form Video Scripts — TikTok (2)
- 15-30 seconds, native feel, pattern interrupt hook in first 1 second
- Vulnerability/confession or controversial opinion hooks
- Soft CTA ("link in bio"), no hard sell
- Must pass the "would I scroll past this?" test

### Pinterest Pins (3)
- Vertical format (1000x1500)
- Lifestyle/aspirational framing of each insight
- Text overlay: 50 chars max
- Optimized for search discovery (include relevant keywords)

## File Organization
Create `repurposed-[source-title]-[date]/`:
- tweets.md
- linkedin.md
- email-sections.md
- ad-hooks.md
- video-scripts-meta.md
- video-scripts-tiktok.md
- pinterest-pins.md
- index.md (summary of everything with one-line descriptions)

## Rules
- Load brand voice file if it exists in project
- Every piece provides standalone value — no "check out my video" filler
- Each asset takes a DIFFERENT angle — no redundancy across outputs
- Platform formatting matters: tweets look like tweets, LinkedIn looks like LinkedIn
- Save everything in project folder under organized structure
