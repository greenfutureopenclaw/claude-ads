# Social Media Creative Operations

## Context Files — Auto-Load
Always load these reference files when executing any creative skill:
- `context/brand-voice.md` — Full brand kit: colors, fonts, tone, visual identity, copy rules
- `context/brief-template.md` — Default creative brief template
- `context/personas.md` — Target audience personas with pain points, emotional triggers, and marketing angles
- `context/ad-formats.json` — Library of 20 ad formats with visual rules and copy guidelines
- `context/shopify/products.md` — Shopify product catalog (auto-synced via `/shopify-sync`)
- `context/shopify/reviews.md` — Customer reviews from Shopify store
- `context/shopify/orders.md` — Sales metrics and order data (last 60 days)
- `context/shopify/segments.md` — Audience micro-segments from RFM analysis (generated by `/audience-segments`)
- `context/google-ads/performance.md` — Google Ads campaign and ad group performance (synced via `/google-ads-sync`)
- `context/tiktok-ads/performance.md` — TikTok Ads campaign and ad-level performance (synced via `/tiktok-sync`)
- `context/meta/performance.md` — Meta (Facebook Page + Instagram) performance data (synced via `/meta-sync`)

## Brand Assets
- `context/assets/` — Logo files, product images, lifestyle photos (JPG, PNG, WebP)
- `context/reference-ads/` — Best-performing ads used as on-brand style references

## Reference Ad Auto-Injection
When any creative skill runs, automatically check `context/reference-ads/` for files. If reference ads exist, describe them to the AI as **"this is what on-brand advertising looks like for this brand"** — use them as a soft style influence on all generated creative. Do not require the user to manually reference them.

## Output Conventions
- Save all outputs as markdown files in the project folder
- Use naming pattern: `[type]-[brand/campaign]-[date].md`
- HTML outputs (static ads, advertorials) go in dated subfolders
- Multi-file outputs (concepts, repurposed content) go in named folders with an index.md

## Social Flow CLI
The `social-flow` package is installed for Meta API operations (posting, ads management, insights). Configuration is in `.env`. Use `social` CLI commands for direct platform interactions.

## Platform Integrations

### Shopify
Synced via `node scripts/sync.js` (or `/shopify-sync`). Requires `SHOPIFY_STORE_DOMAIN` and `SHOPIFY_ACCESS_TOKEN` in `.env`. Data lands in `context/shopify/`.

### Google Ads
Synced via `node scripts/google-ads-sync.js` (or `/google-ads-sync`). Requires Google Ads API credentials in `.env`. Data lands in `context/google-ads/`.

### TikTok Ads
Synced via `node scripts/tiktok-sync.js` (or `/tiktok-sync`). Requires `TIKTOK_ACCESS_TOKEN` and `TIKTOK_ADVERTISER_ID` in `.env`. Data lands in `context/tiktok-ads/`.

### Meta (Facebook + Instagram)
Synced via `node scripts/meta-sync.js` (or `/meta-sync`). Requires `META_ACCESS_TOKEN`, `META_PAGE_ID`, and `META_INSTAGRAM_ID` in `.env`. Data lands in `context/meta/`. Test connection with `npm run meta:test`.

### Google Workspace (Drive, Calendar, Gmail, Reviews)
OAuth setup via `npm run google:setup` (or `/google-drive`, `/google-calendar`, etc.). Requires `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`, and `GOOGLE_REFRESH_TOKEN` in `.env`. Run setup wizard once to generate the refresh token. Context lands in `context/google/`.

### Sync All Platforms
Run `npm run sync:all` to sync Shopify + Google Ads + TikTok + Meta in one command. See `.env.example` for all required credentials.
